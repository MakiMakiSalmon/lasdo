# lasdo

「予定」ではなく **実績** を記録し、自分の **活動リズムをグラフで可視化** するツール。
カレンダーが未来の計画を埋めていくものなら、lasdo は過去の実際の時間の使い方をタイマーで埋めていく。
名前の由来は "last do"（過去にやったこと＝実績の逆引き）。

> 「サボり時間」は `24h − アクティブ時間` で出るおまけにすぎず、主役ではない。
> 力点は **自分のアクティブ/非アクティブのパターンの見える化** にある。

## ステータス

**フェーズ1（MVP・Webコア）実装済み**。記録・分析・編集の3画面が動作する。

- **記録画面**: 円形タイマー（ゼルダBotWのがんばりゲージ風）で計測 → 停止で区間を確定 → 今日のタイムライン（5:00〜翌5:00固定）に即反映。25分単位の外周リング・×N集約・リロード耐性つき。
- **分析画面**: 期間プリセット（直近4週/12週/全期間）に連動して、曜日別の平均活動時間（棒グラフ）と、開始/終了時刻の分布（箱ひげ図・曜日別7箱⇄全体1本）を表示。
- **編集画面**: 区間の追加・時刻変更・削除（重なりは自動マージ）。

> 現時点は **PC 先行**（スマホ UI は後回し）。データ/ロジック/状態は共通なので後から出し分けできる。

## 設計の力点

- **時間を自動分類・自動挿入しない**。保存するのはユーザーが記録したアクティブ区間だけで、非アクティブは「記録の不在」として集計時に算出する。
- データは **開始＋終了の2項目のみ**（種別/カテゴリ/タグは持たない）。
- 深夜またぎ区間は分割せず1ブロックで保存し、日付境界（5:00起点）は表示・集計時にだけ切る。
- 区間の重なりは禁止 → 自動マージ（`アクティブ合計 = 各区間長の総和` が二重計上なく成立）。

## 技術スタック

- **React + Vite + TypeScript** / 状態管理 **Zustand** / テスト **Vitest** / スタイル **CSS Modules**
- 保存は **IndexedDB（Dexie）** を `TimeBlockRepository` の裏に隠す（将来 Supabase へ差し替えてスマホ同期/PCエージェントに対応する想定）。
- 円形タイマーと1日タイムラインは **自前 SVG**、棒/箱ひげは **ECharts**、日付・時刻は **date-fns**。

## 開発

```sh
npm install
npm run dev      # 開発サーバ（http://localhost:5173）
npm test         # Vitest（ドメイン純粋関数が主戦場）
npm run lint     # ESLint
npm run build    # 型チェック + 本番ビルド
```

`main` への直接 push は不可。feature ブランチ → Pull Request → セルフマージで運用する（詳細は [docs/git-workflow.md](docs/git-workflow.md)）。
